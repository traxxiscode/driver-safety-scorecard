ⓘ  How Driver Scoring Works ✕
Score Formula
Every driver starts with a perfect score of 100. Points are deducted for each safety rule violation, scaled by miles driven in the period.
Score = 100 − ∑ Rule Deductions
Rule Deduction = min( Weight, Rate × Points/Incident )
Rate = Events ÷ Miles Driven × 100
Scoring is normalized per 100 miles driven so high-mileage drivers aren't unfairly penalized for volume.
Weights
Each enabled rule has a Weight (0–100) representing its share of total score impact. All weights must sum to exactly 100. When rules change, weights are auto-equalized. Saved to Firestore for backend access.
Points Per Incident
Set per rule in the Rules panel. Controls how many raw penalty points each event generates before the weight cap applies. Higher = more sensitive to individual incidents.

Firestore & Backend
Rule config, weights, and schedule settings are persisted to Cloud Firestore, keyed by Geotab database name. Your AWS EC2 backend polls Firestore to discover which databases have active schedules and which recipients to email — no manual configuration needed on the server side.
Data Access & Security
This add-in uses the authenticated Geotab API session. All driver, vehicle, and exception data is scoped to the logged-in user's group visibility in Geotab. No fleet data is stored externally — only configuration is written to Firestore.
